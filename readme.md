使用vantUI + VUE 生成项目
基于需求，设计包含管理员、社区 / 物业公司、业主三类角色的移动端原型，聚焦核心流程：业主登录上报、管理员审核、物业 / 社区处理，以下是完整原型方案（含页面结构、交互逻辑、核心功能）：
一、原型整体架构
角色	核心流程	关键页面
业主	口令登录 → 查看工单通知 → 报修 / 投诉 → 查看工单进度	登录页、首页、报修页、投诉页、工单详情页
社区 / 物业	账号密码登录 → 查看待处理工单 → 处理工单（录入情况 + 传图） → 提交处理结果	登录页、工单列表页、工单处理页、处理记录页
管理员	账号密码登录 → 审核工单 → 批准 / 驳回 → 查看审核记录	登录页、待审核工单列表、工单审核页、审核记录页
二、核心页面原型（含交互说明）
1. 公共登录页（区分角色）
页面布局：
顶部：App 名称「小区家园通」+ Logo
中间：
角色选择器（下拉框：业主 / 社区 / 物业 / 管理员）
账号输入框（业主：仅「口令」输入；社区 / 物业 / 管理员：账号 + 密码）
登录按钮（蓝色主色调，点击触发验证）
忘记口令 / 密码（仅社区 / 物业 / 管理员显示）
底部：版权信息（如「©2025 小区物业服务中心」）
交互逻辑：
选择「业主」角色：输入框自动切换为「6 位数字口令」（仅数字输入键盘），验证通过直接登录
选择「社区 / 物业」「管理员」角色：显示「账号 + 密码」输入框（账号支持手机号 / 用户名，密码隐藏输入）
登录失败提示：口令 / 账号密码错误（红色文字提示，3 秒自动消失）
2. 业主端核心页面
（1）业主首页
布局：
顶部：欢迎语「欢迎回来，业主 XXX」+ 退出登录按钮
核心功能区（2 个大卡片，图标 + 文字）：
「我要报修」（橙色图标，维修工具样式）
「我要投诉」（红色图标，感叹号样式）
工单进度区：「我的工单」（列表展示，含工单类型、提交时间、当前状态：待审核 / 处理中 / 已完成）
底部：消息通知入口（小红点提示未读）
交互：
登录后自动弹出「最新工单处理情况」弹窗（含工单编号、处理阶段、处理备注，弹窗可关闭 / 查看详情）
点击「我要报修 / 投诉」跳转对应提交页
点击工单列表项跳转「工单详情页」
（2）报修 / 投诉提交页（结构一致，仅字段差异）
报修页字段：
报修类型（下拉选择：水电维修 / 家电维修 / 公共区域维修 / 其他）
所在楼栋（输入框：如「3 栋 2 单元 501」）
报修描述（多行输入框， placeholder：请详细描述问题，至少 10 字）
上传图片（最多 3 张，支持拍照 / 相册选择，预览 + 删除功能）
联系电话（默认填充业主预留手机号，可修改）
提交按钮（底部固定，点击前校验必填项）
投诉页字段：
投诉类型（下拉选择：服务态度 / 卫生问题 / 噪音扰民 / 其他）
其他字段与报修页一致
交互：
必填项（类型、描述、楼栋、电话）未填时，提交按钮置灰并提示「请完善必填信息」
图片上传后显示缩略图，点击缩略图可放大预览，右上角有关闭按钮删除
提交成功后显示「上报成功，等待管理员审核」提示（3 秒消失），自动返回首页
（3）业主工单详情页
布局：
顶部：工单编号（如「BX20250610001」）+ 状态标签（待审核 / 处理中 / 已完成，不同状态不同颜色）
核心信息区：
上报类型、提交时间、所在楼栋、联系电话
问题描述（分行展示）
上传图片（横向滑动预览）
处理进度区（仅审核通过后显示）：
审核通过时间、审核人（管理员匿名）
物业处理时间、处理人员
处理说明（多行文字）
处理图片（物业上传，横向滑动预览）
底部：「刷新进度」按钮（点击更新最新状态）
3. 管理员端核心页面
（1）管理员首页
布局：
顶部：「管理员后台」+ 退出登录按钮
数据概览（3 个统计卡片）：
待审核工单（红色数字，显示数量）
已审核工单（蓝色数字）
已处理工单（绿色数字）
功能入口：
待审核工单列表（优先展示，带小红点提示数量）
审核记录查询
账号管理（可选，用于管理物业账号）
（2）待审核工单列表页
布局：
顶部：搜索框（支持按工单编号、楼栋、业主电话搜索）
列表项（每条包含）：
工单类型（报修 / 投诉，图标区分）
提交时间、所在楼栋
提交人（业主匿名，如「3 栋业主」）
状态标签（「待审核」灰色）
操作按钮（「审核」蓝色按钮，右侧对齐）
交互：
点击「审核」跳转至「工单审核页」
列表支持下拉刷新、上拉加载更多
（3）工单审核页
布局：
顶部：返回按钮 + 「工单审核」标题
工单基础信息（同业主工单详情页的核心信息区）
审核操作区（底部固定）：
审核意见输入框（可选，placeholder：请输入审核意见，如驳回原因）
两个按钮：「批准」（绿色）、「驳回」（红色）
交互：
点击「批准」：
若填写审核意见，同步保存意见
提示「审核通过，已同步至物业端」（3 秒消失）
自动返回待审核列表，该工单从列表移除
点击「驳回」：
强制要求填写审核意见（如「问题描述不清晰，请补充信息」）
提示「驳回成功，已通知业主」（3 秒消失）
业主端会收到驳回消息（首页消息入口小红点提示）
4. 社区 / 物业端核心页面
（1）物业首页
布局：
顶部：「物业工单处理平台」+ 退出登录按钮
数据概览（2 个统计卡片）：
待处理工单（橙色数字，显示数量）
已处理工单（绿色数字）
功能入口：
待处理工单列表（带橙点提示数量）
处理记录查询
（2）待处理工单列表页
布局：
顶部：搜索框（支持按工单编号、楼栋搜索）
列表项（每条包含）：
工单类型（图标区分）
审核通过时间、所在楼栋
状态标签（「待处理」橙色）
操作按钮（「处理」蓝色按钮）
交互：
仅显示管理员审核通过的工单
点击「处理」跳转至「工单处理页」
（3）工单处理页
布局：
顶部：返回按钮 + 「工单处理」标题
工单基础信息（同业主工单详情页核心信息区）
处理操作区：
处理说明（多行输入框，必填，placeholder：请详细描述处理过程及结果）
上传处理图片（最多 3 张，支持拍照 / 相册选择，预览 + 删除）
处理人员（输入框，默认填充当前登录账号姓名，可修改）
提交按钮（底部固定，蓝色）
交互：
处理说明、处理图片为必填项，未填时提交按钮置灰并提示
图片上传逻辑同业主提交页
提交成功后提示「处理完成，已同步至业主端」（3 秒消失），自动返回待处理列表
提交后工单状态更新为「已完成」，业主端可查看处理结果
三、核心交互规则总结
角色权限控制：
业主仅能查看自己的工单，无法查看他人上报内容
物业仅能查看管理员审核通过的工单，无法审核工单
管理员仅能审核工单，无法处理工单
工单流转逻辑：业主提交 → 管理员审核（批准 / 驳回）→ 批准后物业可见并处理 → 处理完成后业主查看结果
消息通知规则：
业主：登录后弹窗最新工单处理情况；工单被驳回 / 审核通过 / 处理完成时，首页消息入口小红点提示
物业：新的待处理工单生成时，首页待处理数量更新并提示
管理员：新的待审核工单生成时，待审核数量更新并提示
